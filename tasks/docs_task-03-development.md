# Задача 3. Разработка: Реализованные фичи

## 1. Цель задачи
Полная программная реализация бэкенд и фронтенд частей приложения MVP в строгом соответствии с архитектурным контрактом. Требуется покрыть кодом базовые и расширенные сценарии использования для всех ролей.

## 2. Ссылка на бизнес-требования
* **R-daily / R-027:** Ежедневная генерация расписания по 3 занятия в день без наложений. Фильтрация на 7/14 дней.
* **R-cap / R-rent:** Автоматическое зашивание ограничений мест (6 для круга, 10 для лепки) и проката (400 ₽) в код.
* **R-004 / R-008 / R-cancel:** Логика атомарного списания мест, проверки отмены студии и правила отсечки отмены клиентом (2 часа).

## 3. Описание реализации фич
Реализовано 7 ключевых фич:
1. **Каталог занятий:** Бэкенд генерирует «живое» расписание на 14 дней. Фронтенд группирует слоты по дням, подсвечивает заполненность (цветовые бары) и фильтрует по программам.
2. **Карточка занятия:** Динамическая сборка данных слота, пересчет итоговой стоимости при выборе проката, скрытие формы при недоступности записи.
3. **Бронирование:** POST-запрос с защитой от race condition, возвращающий 201 при успехе или 409 при конфликте мест. Анимированный экран успеха (`success.html`).
4. **Мои записи и отмена:** Поиск броней по нормализованному номеру телефона. Отмена с проверкой `start_time - теперь >= 2 часов`.
5. **Витрина мастеров:** Сетка мастеров с сортировкой по убыванию рейтинга.
6. **Аутентификация:** Bearer Token авторизация пользователей с разделением прав доступа.
7. **Панель администратора:** Полнофункциональный интерфейс для владельца, включающий инлайн-правку параметров слота (`PATCH`), отмену занятия (`POST /cancel`) и его восстановление.

На фронте — карточки + empty state, экран деталей с прокатом, отправка формы с редиректом на success при 201 и ошибкой при 409. Дай код и коммит-месседжи.`

Переработай CSS в дизайн-систему (керамическая палитра, hero, карточки с индикатором заполненности, итоговая цена на кнопке) и сделай адаптив под телефон и десктоп.`
