## 2. Найденные баги и исправления

### Баг #1 — переполнение слота: бронь заполненного занятия проходит (логический)

* **Симптом:** запрос брони на слот, где `remaining_places = 0`, возвращал `201`
  и уводил `booked_count` за пределы `capacity` (двойная бронь).
* **Бизнес-требование:** R-004 — атомарная проверка мест, при отсутствии → 409.
* **Причина:** проверка использовала `remaining < 0` вместо `remaining <= 0`,
  поэтому последнее «лишнее» бронирование проскакивало.
* **Тест-кейс:** TC-09, TC-11.

### Баг #2 — запись на отменённое мастерской занятие (логический)

* **Симптом:** бронь слота со статусом `CANCELLED_BY_STUDIO` возвращала `201`.
* **Бизнес-требование:** R-008 — повторная запись на отменённый слот запрещена.
* **Причина:** в `create_booking` отсутствовала проверка статуса слота перед
  созданием брони.
* **Тест-кейс:** TC-10.

**Единый диф по бэкенду (`backend/api.py`):**
```diff
         slot = SLOTS.get(slot_id)
         if slot is None:
             return jsonify({"error": "slot not found"}), 404

+        # R-008: запрет записи на слот, отменённый мастерской.
+        if slot["status"] == STATUS_CANCELLED_BY_STUDIO:
+            return jsonify({
+                "error": "slot_cancelled",
+                "message": "Занятие отменено мастерской. Запись недоступна.",
+                "cancel_reason": slot["cancel_reason"],
+            }), 409
+
         remaining = slot["capacity"] - slot["booked_count"]
-        # BUG: проверка использует `< 0` вместо `<= 0` ...
-        if remaining < 0:
+        # R-004: нет свободных мест — отклоняем бронь.
+        if remaining <= 0:
             return jsonify({
                 "error": "no_places",
                 "message": "Мест не осталось",
-                "remaining_places": max(remaining, 0),
+                "remaining_places": 0,
             }), 409
```

### Баг #3 — тупиковая кнопка «Записаться» на заполненном слоте (UI)

* **Симптом:** в расписании полностью заполненный слот (`remaining = 0`, но не
  отменён) показывал активную зелёную кнопку «Записаться». Клик вёл в карточку,
  где запись всё равно недоступна — тупиковый путь.
* **Бизнес-требование:** US-A4 (визуально отличать заполненные занятия).
* **Причина:** `renderSlotCard` учитывал только флаг `cancelled`, но не `full`.
* **Тест-кейс:** TC-14.

**Диф по фронтенду (`frontend/app.js`):**
```diff
-  const btn = document.createElement('a');
-  btn.className = 'btn btn--primary card__btn';
-  btn.href = `details.html?id=${slot.id}`;
-  btn.textContent = cancelled ? 'Подробнее' : 'Записаться';
-  card.appendChild(btn);
+  if (full && !cancelled) {
+    // Мест нет: не ведём пользователя в тупик, показываем неактивную кнопку.
+    const btn = document.createElement('span');
+    btn.className = 'btn btn--disabled card__btn';
+    btn.textContent = 'Мест нет';
+    card.appendChild(btn);
+  } else {
+    const btn = document.createElement('a');
+    btn.className = 'btn btn--primary card__btn';
+    btn.href = `details.html?id=${slot.id}`;
+    btn.textContent = cancelled ? 'Подробнее' : 'Записаться';

+  }
```
