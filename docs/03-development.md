# Этап 3. Разработка: реализация ключевых фич

> Отчёт о реализации 3 ключевых фич MVP. Опирается на контракт из
> `docs/02-design.md`. Код: `backend/api.py`, `frontend/*`.

## Фича 1 — Каталог занятий (Расписание)

**Бизнес-требование:** US-A1…A4, R-027 (горизонт 7 дней), empty state.

**Бэкенд (`backend/api.py`):**
* `GET /api/slots` — фильтрует `SLOTS` по периоду. По умолчанию — `[сегодня; +7 дней]`
  (R-027), период расширяется параметрами `date_from` / `date_to`.
* Прошедшие слоты исключаются (`start < now`).
* Результат сортируется по времени начала; отдаётся `{ "slots": [...], "count": N }`.
* Каждый слот разворачивается `serialize_slot()` — добавляются данные программы,
  мастера, `remaining_places`, `is_bookable`.

**Фронтенд (`frontend/index.html`, `app.js → initSchedule`):**
* Загружает слоты, рендерит карточки (`renderSlotCard`): программа, дата/время,
  мастер с рейтингом, цена, бейдж статуса (свободно / мест нет / отменено).
* Если `slots.length === 0` — показывается блок **empty state**
  («Пока нет доступных занятий»).
* Ошибка сети → alert «Проверьте, что сервер запущен».

**Коммит:** `feat(frontend): расписание занятий с фильтром 7 дней и empty state`

## Фича 2 — Карточка занятия

**Бизнес-требование:** US-B1…B3, прокат без размеров, отображение отмены (R-008).

**Бэкенд:**
* `GET /api/slots/{id}` — возвращает полный объект слота или `404`.

**Фронтенд (`frontend/details.html`, `app.js → initDetails`):**
* По `?id=` грузит слот, рендерит детали: описание программы, длительность,
  вместимость (до 6 / до 10), рейтинг мастера, число мест, **прокат** (цена без
  размеров), адрес мастерской.
* Если слот `CANCELLED_BY_STUDIO` — форма скрыта, показывается причина отмены.
* Если мест нет — форма скрыта, сообщение «Все места заняты».

**Коммит:** `feat(frontend): карточка занятия с деталями и формой записи`

## Фича 3 — Бронирование

**Бизнес-требование:** US-C1…C4, R-004 (атомарность, 409), R-008.

**Бэкенд (`app.py → create_booking`):**
* `POST /api/bookings` — валидирует `slot_id`, `customer_name`, `customer_phone`.
* Критическая секция под `threading.Lock`: проверка слота → проверка мест →
  инкремент `booked_count` → запись брони. Возвращает `201` с бронью и слотом.
* Отсутствие мест → `409` (R-004). Отменённый слот → отклоняется (R-008).

**Фронтенд (`app.js → submitBooking`):**
* Собирает данные формы (имя, телефон, чекбокс проката), POST-ит бронь.
* `201` → редирект на `success.html` с параметрами (имя, программа, время, прокат).
* `409` → сообщение «Мест не осталось», без перехода.
* `success.html` (`initSuccess`) — экран подтверждения с деталями записи и
  напоминанием об оплате на месте.

**Коммит:** `feat(backend): транзакционное бронирование слотов (R-004)`

## Как запустить

```bash
# Backend
cd backend
pip install -r requirements.txt
python api.py            # http://127.0.0.1:5000

# Frontend — открыть frontend/index.html в браузере
# (или обслуживать статикой: python -m http.server 5500 из папки frontend)
```

## Приложение. Использованные промпты

**Промпт 1 (каталог):**
> Реализуй на Flask эндпоинт GET /api/slots: фильтр по датам с дефолтом 7 дней
> (R-027), исключение прошедших слотов, сортировка по времени, обёртка каждого
> слота с данными программы/мастера и полем remaining_places. На фронте — рендер
> карточек и empty state при пустом списке. Дай код бэкенда и фронта + коммит-месседж.

**Промпт 2 (карточка):**
> Добавь GET /api/slots/{id} и экран details.html: детали занятия, прокат без
> размеров, адрес мастерской. Если слот CANCELLED_BY_STUDIO или мест нет — скрывай
> форму записи и показывай причину. Дай код и коммит-месседж.

**Промпт 3 (бронирование):**
> Реализуй POST /api/bookings с атомарной секцией под threading.Lock: проверка
> мест (R-004, 409 при отсутствии) и запись брони. На фронте — отправка формы,
> редирект на success.html при 201 и показ ошибки при 409. Дай код и коммит-месседж.
