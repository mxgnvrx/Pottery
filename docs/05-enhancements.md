# Этап 5. Расширение MVP: новые фичи и редизайн

> Отчёт о доработке продукта после сдачи базового MVP.
> Ссылки: бриф — `docs/brief-pottery.md`, базовые фичи — `docs/03-development.md`.

## 1. Цель

Базовый MVP (3 экрана) закрывал только happy-path записи. Доработка добавляет
сценарии из брифа Марины, которые не вошли в первую поставку, и полноценный
визуальный дизайн.

## 2. Новые фичи

### 2.1. «Мои записи» + отмена брони клиентом (`bookings.html`)

**Бизнес-требование (бриф):** «Иногда люди отменяются. Если заранее — ок, место
освобождается. Если за 10 минут до начала — это уже проблема… Надо как-то это
ограничить, но чтобы клиентов не злить».

* `GET /api/bookings?phone=…` — все брони по номеру телефона (нормализация до
  последних 10 цифр, чтобы `+7 900 000-00-00` и `89000000000` совпадали).
* `POST /api/bookings/{id}/cancel` — отмена брони клиентом:
  * **правило отсечки** — не позднее чем за **2 часа** до начала
    (`CANCEL_CUTOFF_HOURS`), иначе `409 late_cancellation` с вежливым
    сообщением «позвоните нам»;
  * место освобождается (`booked_count -= 1`) — атомарно, под тем же
    `_booking_lock`, что и бронирование;
  * бронь по слоту `CANCELLED_BY_STUDIO` отменять нельзя (`409`) — она уже
    помечена статусом слота (R-008), клиент видит причину отмены.
* Экран: поиск по телефону (номер запоминается в `localStorage` после брони),
  карточки записей со статусами: «Подтверждена», «Отменена вами»,
  «Отменено мастерской» (+причина), «Прошло».

### 2.2. «Мастера» (`masters.html`)

**Бизнес-требование (бриф):** «Хочу, чтобы клиент мог поставить оценку мастеру…
есть пара ребят-звёзд, к ним очередь» — витрина рейтингов (выставление оценок
после занятия — следующая итерация, вне текущего скоупа).

* `GET /api/masters` — команда с рейтингами, специализацией и биографией,
  отсортирована по рейтингу.
* Экран: сетка карточек — аватар, специализация, звёзды рейтинга, био.

### 2.3. Фильтры расписания (`index.html`)

**Бизнес-требование:** R-027 — «более длинный период доступен через фильтр дат».

* Чипы-фильтры по программе: Все / Гончарный круг / Лепка руками (клиентская
  фильтрация).
* Кнопка «Показать 14 дней» — расширяет горизонт через `?date_to=` (дефолт
  остаётся 7 дней, R-027).
* Расписание сгруппировано по дням: «Сегодня», «Завтра», «пятница, 10 июля».

## 3. Редизайн

Полностью переработан `style.css` — дизайн-система в тёплой «керамической» гамме:

* **Типографика:** Playfair Display (заголовки) + Manrope (текст), Google Fonts.
* **Палитра:** терракота `#b45a33`, крем `#f6efe4`, тёмная умбра `#2c211a`;
  смысловые цвета статусов (зелёный/янтарный/красный) с мягкими подложками.
* **Компоненты:** hero-блок с бейджем и статистикой; липкая полупрозрачная шапка
  с навигацией (blur); карточки с иконкой программы, крупным временем и
  **индикатором заполненности** (полоска: зелёная → янтарная при ≤2 местах →
  красная при 0); чипы-фильтры; футер с адресом.
* **Карточка занятия:** сетка фактов (длительность/группа/мастер/места), прокат
  оформлен как опция с ценой, кнопка показывает **итоговую сумму** и
  пересчитывается при выборе проката.
* **Экран успеха:** анимированная галочка (CSS keyframes), сводка брони,
  подсказка про отмену за 2 часа, переходы в «Мои записи» и расписание.
* Mobile-first, адаптив до 560px (клиенты Марины — «в основном с телефона»).

## 4. Демо-данные

Для проверки «Моих записей» посеяны 2 брони на телефон `+7 900 000-00-00`:
подтверждённая (можно отменить) и бронь на слот, отменённый мастерской
(видна причина «сломалась печь», отмена заблокирована).

## 5. Проверка

| Сценарий | Ожидание | Результат |
|----------|----------|-----------|
| `GET /api/masters` | 4 мастера по убыванию рейтинга | ✅ |
| `GET /api/bookings?phone=+79000000000` | 2 демо-брони | ✅ |
| `GET /api/bookings?phone=123` | 400 | ✅ |
| Отмена будущей брони (>2ч) | 200, место освобождено (1→2) | ✅ |
| Отмена брони <2ч до начала | 409 `late_cancellation` | ✅ |
| Отмена брони на отменённом слоте | 409 `slot_cancelled` | ✅ |
| Повторная отмена | 409 `already_cancelled` | ✅ |
| Все 5 страниц отдаются | 200 | ✅ |

**Коммиты:**
* `feat(backend): мои записи, отмена брони (правило 2 часов), мастера`
* `feat(frontend): редизайн + страницы «Мастера» и «Мои записи»`

## Приложение. Использованные промпты

**Промпт 1 (постановка):**
> Усложни проект: 3 страницы — мало. Добавь сценарии из брифа, не вошедшие в MVP
> (отмена брони клиентом с освобождением места, витрина мастеров с рейтингами,
> фильтры расписания), и сделай красивый дизайн.

**Промпт 2 (отмена брони):**
> Реализуй GET /api/bookings?phone= и POST /api/bookings/{id}/cancel: нормализация
> телефона, правило отсечки за 2 часа (бриф: поздние отмены — проблема, но клиентов
> не злить), освобождение места под тем же lock, что и бронирование, запрет отмены
> по слотам CANCELLED_BY_STUDIO. Экран «Мои записи» со статусами броней.

**Промпт 3 (мастера):**
> Добавь GET /api/masters (специализация, био, рейтинг, сортировка по рейтингу)
> и страницу-витрину мастеров с аватарами и звёздами.

**Промпт 4 (редизайн):**
> Переработай CSS в дизайн-систему: тёплая керамическая палитра (терракота/крем),
> Playfair Display + Manrope, hero-блок, липкая шапка с навигацией, карточки с
> индикатором заполненности, итоговая цена на кнопке с пересчётом проката,
> анимированный экран успеха, футер, mobile-first адаптив.
