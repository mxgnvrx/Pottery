## Фича 1 — Каталог занятий (расписание)

**Бэкенд:** `GET /api/slots` фильтрует `SLOTS` по периоду (дефолт `[сегодня; +7 дней]`,
R-027), исключает прошедшие слоты, сортирует по времени, разворачивает каждый слот
`serialize_slot()` (программа, мастер, `remaining_places`, `is_bookable`).

**Расписание — ежедневно (R-daily):** `seed_data` генерирует занятия на 14 дней
вперёд, **по 3 сеанса в день** (11:00 / 15:00 / 19:00; длительность 2–2.5 ч, окна
не пересекаются — ресурсы 4 мастера / 10 кругов не превышаются). Мастера
назначаются по специальности, заполненность разнообразная, сохранены демо-состояния
(полный / почти полный / свободный / отменённый).

**Фронтенд** (`index.html`, `initSchedule`): карточки с программой, датой/временем,
мастером и рейтингом, ценой, **индикатором заполненности**; группировка по дням
(«Сегодня», «Завтра», «пятница, 10 июля»); чипы-фильтры по программе; кнопка
«Показать 14 дней» (расширение горизонта через `?date_to=`); **empty state** при
пустом списке.

**Коммит:** `feat(frontend): расписание занятий с фильтром 7 дней и empty state`

## Фича 2 — Карточка занятия

**Бэкенд:** `GET /api/slots/{id}` — полный объект слота или `404`.

**Фронтенд** (`details.html`, `initDetails`): описание программы, длительность,
вместимость (до 6 / до 10), рейтинг мастера, число мест, **прокат** (цена без
размеров), адрес мастерской. Кнопка показывает **итоговую сумму** с пересчётом при
выборе проката. Если слот `CANCELLED_BY_STUDIO` — форма скрыта, показана причина;
если мест нет — форма скрыта, «Все места заняты».

**Коммит:** `feat(frontend): карточка занятия с деталями и формой записи`

## Фича 3 — Бронирование

**Бэкенд** (`create_booking`): `POST /api/bookings` валидирует поля; критическая
секция под `threading.Lock` — проверка статуса слота (R-008) → проверка мест
(R-004) → инкремент `booked_count` → запись брони → `201`. Нет мест → `409`
`no_places`; отменённый слот → `409` `slot_cancelled`.

**Фронтенд** (`submitBooking`): POST брони; `201` → редирект на `success.html` с
деталями; `409` → «Мест не осталось» без перехода. `success.html` — экран
подтверждения с анимированной галочкой, сводкой и напоминанием об оплате на месте.

**Коммит:** `feat(backend): транзакционное бронирование слотов (R-004)`

## Фича 4 — Мои записи + отмена брони клиентом

**Бизнес-требование (бриф):** «Иногда люди отменяются. Если заранее — ок… если за
10 минут до начала — проблема. Надо ограничить, но клиентов не злить.»

**Бэкенд:**
* `GET /api/bookings?phone=…` — брони по телефону (нормализация до последних 10
  цифр, чтобы `+7 900 000-00-00` и `89000000000` совпадали).
* `POST /api/bookings/{id}/cancel` — отмена клиентом: **правило отсечки за 2 часа**
  (`CANCEL_CUTOFF_HOURS`), иначе `409 late_cancellation` («позвоните нам»); место
  освобождается под тем же `_booking_lock`; бронь по слоту `CANCELLED_BY_STUDIO`
  отменять нельзя (`409`).

**Фронтенд** (`bookings.html`): поиск по телефону (номер запоминается в
`localStorage`), карточки со статусами «Подтверждена» / «Отменена вами» /
«Отменено мастерской» (+причина) / «Прошло».

**Коммит:** `feat(backend): мои записи, отмена брони (правило 2 часов)`

## Фича 5 — Витрина мастеров

**Бэкенд:** `GET /api/masters` — команда с рейтингами, специализацией и биографией,
отсортирована по рейтингу.

**Фронтенд** (`masters.html`): сетка карточек — аватар, специализация, звёзды
рейтинга, био. (Выставление оценок после занятия — следующая итерация.)

**Коммит:** `feat(frontend): страница «Мастера»`

## Фича 6 — Вход и роли

**Бэкенд:** in-memory пользователи (`admin/admin123`, `client/client123`);
`POST /api/auth/login` → `{token, user}` или `401`; `GET /api/auth/me` — проверка
токена; токен в `Authorization: Bearer <token>`; админ-роуты закрыты проверкой
роли (`403`).

**Фронтенд** (`login.html`): форма логин/пароль + быстрые кнопки демо-аккаунтов.
После входа: администратор → панель, клиент → «Мои записи» (телефон подставляется).
В шапке появляются «Выйти» и, для админа, ссылка «Панель администратора».

**Коммит:** `feat(backend): аутентификация и роли`

---

## Фича 7 — Панель администратора

**Бизнес-требование:** из письма Марины — «экран, где я вижу всё расписание и могу
что-то поправить руками, если что-то пошло не так» (US-F1, US-F2).

**Бэкенд:**
* `GET /api/admin/slots` — всё расписание (вкл. прошедшие и отменённые) + состав
  броней на каждый слот.
* `POST /api/admin/slots/{id}/cancel` `{reason}` — отменить занятие (R-008):
  статус `CANCELLED_BY_STUDIO`, причина видна клиентам, брони сохраняются.
* `POST /api/admin/slots/{id}/restore` — вернуть ошибочно отменённое занятие.
* `PATCH /api/admin/slots/{id}` — **ручная правка**: число мест (`capacity`),
  **мастер** (`master_id`, напр. заболел), **дата/время** (`start_time`, перенос),
  длительность (`duration_min`), с валидацией (см. `02-architecture.md`).

**Фронтенд** (`admin.html`): список всех занятий со сводкой (слотов / записей /
отменено); на каждой строке — степпер числа мест, инлайн-форма правки (дата, время,
мастер) с «Сохранить», кнопки «Отменить занятие» / «Вернуть в расписание».

**Коммит:** `feat: панель администратора — просмотр и ручная правка занятий`

## Сквозные доработки

**Редизайн** (`style.css`): дизайн-система в тёплой керамической гамме — терракота
`#b45a33`, крем `#f6efe4`, умбра `#2c211a`; Playfair Display + Manrope; hero-блок,
липкая шапка с blur, карточки с индикатором заполненности, чипы-фильтры,
анимированный экран успеха, футер с адресом.

**Адаптив:** mobile-first (клиенты Марины «в основном с телефона») + брейкпоинты
≥ 900px (контейнер 1120px, hero в две колонки, сетки 3–4 колонки) и ≥ 1280px.

**Демо-данные:** брони на телефон `+7 900 000-00-00` для проверки «Моих записей»
(подтверждённая + на отменённом слоте) — привязываются к реальным будущим слотам.

**Коммиты:** `feat(frontend): редизайн + десктопный адаптив`

## Как запустить

```bash
# Backend (Windows: используйте py вместо python)
cd backend
py api.py                 # http://127.0.0.1:5000

# Frontend — открыть frontend/index.html
# (или py -m http.server 5500 из папки frontend)
```
Быстрый старт под Windows — `backend/run.bat` и `frontend/run.bat`.

## Приложение. Использованные промпты

**Промпт 1 (базовые фичи — каталог/карточка/бронь):**
> Реализуй на Flask: GET /api/slots (фильтр дат, дефолт 7 дней R-027, исключение
> прошедших, remaining_places), GET /api/slots/{id}, POST /api/bookings с атомарной
> секцией под threading.Lock (R-004, 409 при отсутствии мест; R-008 для отменённых).
> На фронте — карточки + empty state, экран деталей с прокатом, отправка формы с
> редиректом на success при 201 и ошибкой при 409. Дай код и коммит-месседжи.

**Промпт 2 (мои записи, отмена, мастера, редизайн):**
> Добавь GET /api/bookings?phone= и POST /api/bookings/{id}/cancel (нормализация
> телефона, правило 2 часов, освобождение места под тем же lock, запрет отмены по
> CANCELLED_BY_STUDIO), GET /api/masters, экраны «Мои записи» и «Мастера».
> Переработай CSS в дизайн-систему (керамическая палитра, hero, карточки с
> индикатором заполненности, итоговая цена на кнопке) и сделай адаптив под телефон
> и десктоп.

**Промпт 3 (вход и панель администратора):**
> Добавь доступ администратора: видит всё расписание и может поправить руками, если
> что-то пошло не так. Сделай логин (админ и клиент), бэкенд-аутентификацию (токен,
> проверка роли) и админ-роуты: просмотр всех слотов, отмена/восстановление (R-008),
> ручная правка числа мест, а также перенос времени и смену мастера.

**Промпт 4 (ежедневное расписание):**
> Скорректируй расписание: студия работает ежедневно, по 3 занятия в день; время
> 2–2.5 ч, 4 мастера, 10 кругов. Показ по дням — ровно 7 дней в дефолтном горизонте.
