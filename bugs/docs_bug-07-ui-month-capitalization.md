# Баг 07. Некорректная капитализация месяцев в датах через CSS

## 1. Симптом бага
Названия месяцев отображались с заглавной буквы («Пятница, 10 Июля»), что нарушает правила русского языка.

## 2. Ссылка на бизнес-требование
* **Требование:** Типографика

## 3. Причина возникновения и Реализация исправления
* **Причина:** Использование CSS-свойства `text-transform: capitalize` для текстовых контейнеров дат, принудительно изменявшего регистр каждого слова.
* **Фикс:** Удалено свойство `text-transform` из `style.css`. Капитализация первой буквы дня недели перенесена в JS-функцию `dayLabel()`.

## 4. Отправленные промпты
> В русском языке названия месяцев пишутся со строчной. CSS капитализирует всё. Убери text-transform и сделай капитализацию первой буквы в JS.

* **Статус:** Исправлено. Поведение системы верифицировано вручную, баг полностью устранен и не воспроизводится.
