# Баг 09. Панель администратора не загружала расписание (Блокирующий CORS)

## 1. Симптом бага
Интерфейс `admin.html` выводил ошибку сети «Не удалось загрузить расписание», хотя бэкенд работал штатно.

## 2. Ссылка на бизнес-требование
* **Требование:** R-028 / CORS

## 3. Причина возникновения и Реализация исправления
* **Причина:** Заголовок CORS `Access-Control-Allow-Headers` на бэкенде принимал только `Content-Type`. Браузер блокировал preflight-запросы `OPTIONS` из-за наличия заголовка авторизации `Authorization: Bearer <token>`.
* **Фикс:** В функцию `add_cors_headers` добавлены методы `PATCH` и разрешен заголовок `Authorization` в `Access-Control-Allow-Headers`.

## 4. Отправленные промпты
> Админка выдает сетевую ошибку при попытке загрузить слоты. Прямой запрос в API возвращает 200. Исправь заголовки CORS-preflight на бэкенде для Authorization.

## 5. Результат ручной проверки и Commit
* **Статус:** Исправлено. Поведение системы верифицировано вручную, баг полностью устранен и не воспроизводится.
* **Commit:** `fix(cors): разрешить Authorization; убрать нерабочую кнопку формы правки`
