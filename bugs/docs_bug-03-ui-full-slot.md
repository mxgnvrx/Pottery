# Баг 03. Тупиковая кнопка «Записаться» на заполненном слоте в каталоге

## 1. Симптом бага
Полностью заполненный слот отображал активную кнопку «Записаться», перенаправляя пользователя на форму, где запись заблокирована.

## 2. Ссылка на бизнес-требование
* **Требование:** US-A4

## 3. Причина возникновения и Реализация исправления
* **Причина:** Функция `renderSlotCard` на фронтенде проверяла только флаг отмены занятия, игнорируя нулевое количество мест.
* **Фикс:** В `app.js` добавлен расчет `full = slot.remaining_places <= 0`. При его истинности рендерится неактивная кнопка с текстом «Мест нет».

## 4. Отправленные промпты
> Заполни слот до максимума на экране расписания. Кнопка остается активной. Исправь renderSlotCard, чтобы кнопка становилась неактивной.
